http-mcp · the wire

A call. A channel.

Two physics, not one. Don't read the difference — fire it, and watch the shapes diverge.

request → response · a call: you ask once, one answer, then silence command ⇄ reply · matched by id on the held line event · arrives unbidden — the afferent half

a call

Request, response, done. The socket opens for one round trip and closes. State lives nowhere between calls — POST /session, an answer, gone. Fire ten; each is alone.

a channel

One socket, held open. You send commands and a reply comes back matched by id — but events arrive on their own, unasked. Close it and every subscription, every sessionId, dies with the line.

Two physics — a call, a channel — and no third smuggled in. · inspired by pretext — experienced, not narrated.
